    üéØ Creating Your First dbt Project¶

    Now that your environment is configured, let's create your first dbt project! We'll use the Jaffle Shop project as an example.

    üöÄ Initializing the Project¶

    1. Create the Project¶

    # Navigate to the directory where you want to create the project
cd ~/Projects

# Initialize new dbt project
dbt init jaffle_shop

# Navigate to the project
cd jaffle_shop

    2. Initial Structure¶

    After initialization, you'll have:

    jaffle_shop/

    ‚öôÔ∏è Project Configuration¶

    1. dbt_project.yml¶

    Edit the dbt_project.yml file:

    name: 'jaffle_shop'
version: '1.0.0'
config-version: 2

profile: 'jaffle_shop'

model-paths: ["models"]
analysis-paths: ["analyses"]
test-paths: ["tests"]
seed-paths: ["seeds"]
macro-paths: ["macros"]
snapshot-paths: ["snapshots"]

target-path: "target"
clean-targets:
    - "target"
    - "dbt_packages"

models:
  jaffle_shop:
    staging:
      +materialized: view
    marts:
      +materialized: table
    analytics:
      +materialized: table

seeds:
  jaffle_shop:
    +column_types:
      id: integer
      customer_id: integer
      order_id: integer
      product_id: integer

    2. profiles.yml¶

    Configure the database connection in ~/.dbt/profiles.yml:

    jaffle_shop:
  target: dev
  outputs:
    dev:
      type: sqlite
      path: "{{ env_var('DBT_SQLITE_PATH', '/tmp/jaffle_shop.db') }}"
      threads: 1

    üóÑÔ∏è Preparing the Data¶

    1. Create SQLite Database¶

    # Create SQLite database
sqlite3 /tmp/jaffle_shop.db

# In SQLite shell, create tables
CREATE TABLE raw_customers (
    id INTEGER PRIMARY KEY,
    first_name TEXT,
    last_name TEXT,
    email TEXT,
    phone TEXT
);

CREATE TABLE raw_products (
    id INTEGER PRIMARY KEY,
    name TEXT,
    category TEXT,
    price REAL
);

CREATE TABLE raw_orders (
    id INTEGER PRIMARY KEY,
    customer_id INTEGER,
    order_date DATE,
    status TEXT
);

CREATE TABLE raw_items (
    id INTEGER PRIMARY KEY,
    order_id INTEGER,
    product_id INTEGER,
    quantity INTEGER,
    unit_price REAL
);

.exit

    üèóÔ∏è Creating Models¶

    1. Staging Model¶

    Create models/staging/stg_customers.sql:

    {{
  config(
    materialized='view'
  )
}}

SELECT
    id,
    first_name,
    last_name,
    email,
    phone,
    -- Clean and standardize data
    TRIM(first_name) as first_name_clean,
    TRIM(last_name) as last_name_clean,
    LOWER(email) as email_clean,
    -- Validate email
    CASE 
        WHEN email LIKE '%@%' THEN 'valid'
        ELSE 'invalid'
    END as email_status
FROM {{ source('raw', 'raw_customers') }}

    2. Mart Model¶

    Create models/marts/customer_summary.sql:

    {{
  config(
    materialized='table'
  )
}}

WITH customer_orders AS (
    SELECT
        c.id as customer_id,
        c.first_name_clean,
        c.last_name_clean,
        c.email_clean,
        COUNT(o.id) as total_orders,
        SUM(CASE WHEN o.status = 'completed' THEN 1 ELSE 0 END) as completed_orders,
        SUM(CASE WHEN o.status = 'pending' THEN 1 ELSE 0 END) as pending_orders
    FROM {{ ref('stg_customers') }} c
    LEFT JOIN {{ ref('stg_orders') }} o ON c.id = o.customer_id
    GROUP BY 1, 2, 3, 4
)

SELECT
    customer_id,
    first_name_clean,
    last_name_clean,
    email_clean,
    total_orders,
    completed_orders,
    pending_orders,
    -- Calculate metrics
    CASE 
        WHEN total_orders > 0 THEN ROUND(completed_orders * 100.0 / total_orders, 2)
        ELSE 0
    END as completion_rate
FROM customer_orders

    üß™ Running the Project¶

    1. Verify Configuration¶

    # Verify everything is configured
dbt debug

    2. Run Models¶

    # Run all models
dbt run

# Run only staging
dbt run --select staging

# Run only marts
dbt run --select marts

    3. Run Tests¶

    # Run all tests
dbt test

# Run specific tests
dbt test --select stg_customers

    4. Generate Documentation¶

    # Generate documentation
dbt docs generate

# Serve documentation locally
dbt docs serve

    üìä Checking Results¶

    1. Query Models¶

    # Check staging
sqlite3 /tmp/jaffle_shop.db "SELECT * FROM stg_customers;"

# Check marts
sqlite3 /tmp/jaffle_shop.db "SELECT * FROM customer_summary;"

    2. Check DAG¶

    # Generate dependency graph
dbt ls --select +customer_summary
